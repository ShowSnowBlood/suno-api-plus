# Product

## Register

product

## Users

Suno API 的部署者和日常运营人员。他们主要在桌面端管理多账号池、查看积分与健康度、提交音乐生成任务，并维护验证码和 OpenAI 兼容接口鉴权；移动端用于快速检查状态和处理轻量配置。

## Product Purpose

提供一个集中、可扫描的运维后台，让管理员能快速判断服务是否可用、哪个账号池有容量、任务是否成功，以及当前接口与验证码配置是否完整。成功标准是常用操作在少量点击内完成，关键异常无需翻找即可发现。

## Brand Personality

清晰、可靠、克制。界面应像成熟的运营控制台，信息密度适中，状态表达直接，不使用营销式装饰。

## Anti-references

- 不做营销落地页式的大标题、渐变背景和装饰性卡片墙。
- 不把所有功能塞进同一个长页面，也不使用含糊的导航名称。
- 不照搬 CPA Manager Plus 的品牌资产，只参考其后台导航密度、分组方式和操作层级。
- 不依赖颜色单独表达状态，不使用难以阅读的低对比度文本。

## Design Principles

1. 先呈现运行状态，再呈现配置细节。
2. 按用户任务拆分模块，生成、任务、账号和安全配置各自独立。
3. 高频操作保持可见，低频说明和高级配置逐步展开。
4. 表格用于比较，卡片只用于独立摘要或明确分组。
5. 桌面端优先高效扫描，移动端保持完整操作能力。

## Accessibility & Inclusion

以 WCAG 2.1 AA 为基线，确保键盘可操作、焦点清晰、正文对比度达标、状态不只依赖颜色，并尊重减少动态效果偏好。
